## 知识点

1. 核心概念

RDMA（Remote Direct Memory Access）本质，是绕过操作系统内核协议栈，网卡直接读写远端服务器的用户态内存，实现**零拷贝、低时延、高带宽、极低 CPU 占用**


2. 区分DMA

传统DMA（2000年在消费级设备普及）只是为了本地设备直接访问内核空间，比如硬盘的DMA控制器可以将数据直接写入内核空间以减少CPU开销，DMA+splice方式真正实现了硬盘数据通过网卡发送的CPU零拷贝,不过是基于TCP，而且splice（Linux2.6引入)方式有一定局限，总之，**核心目的**是为了降低CPU的搬运开销。而且DMA+splice 是基于**传统 TCP/IP 内核协议栈**的CPU零拷贝方案；

RDMA 则是**内核旁路（Kernel Bypass）硬件卸载**offload方案，底层 DMA、内存模型、API、数据通路全部不一样

DMA，作为一个缩写，只是一种方式的描述，并不是一个排他性的名称。RDMA中的DMA是为了**核心目的**高速网络通信采取的方法, 降低CPU占用是手段而不是主要目的.


3. RDMA定位

**硬件 RNIC 网卡独立 DMA，内核旁路，直接操作用户态注册内存，完全绕开 Linux 内核协议栈与管道** 

核心流程（RDMA Write 单边写）： 应用提前注册 MR 内存（锁定物理页，网卡持有虚拟→物理地址映射）→ 用户态下发 WR 工作请求到 QP 队列 → RNIC 硬件 DMA 直接读**用户态内存**封包 → 网络传输 → 对端 RNIC 硬件 DMA 直接写入对端**用户态内存**

1. 硬件直接访问**用户态注册内存 MR**，绕过内核，不需要内核缓冲区做中转站。本地网卡 DMA 直接读用户内存，远端网卡 DMA 直接写用户内存，中间无任何内核缓存中转。
2. 靠 **MR 内存注册、LKEY/RKEY 密钥、网卡内置 MTT 地址翻译表**，内核不参与数据通路，只负责权限校验； RDMA 必须 pin 内存（禁止 swap），和内核页缓存机制完全隔离。
3. 通信单元是 **QP 队列对**
4. RDMA Verbs：用户态库，直接操作网卡硬件队列，**无系统调用、无上下文切换**（数据路径）。


假如要将硬盘的F文件，从A的电脑发给B的电脑.

## 三大主流传输实现

1. **InfiniBand（IB）** 专用存储 / 超算硬件，原生无损，性能最优，成本高，机房私有场景。
2. **RoCE（RDMA over Converged Ethernet）** 以太网承载，数据中心主流：
    - RoCEv1：二层，仅单机架；
    - RoCEv2：UDP/IP，可跨三层，当前 AI 智算集群标配； 短板：默认 GBN 回退重传，微小丢包即吞吐雪崩，依赖 PFC 无损网络。
3. **iWARP（RDMA over TCP）** 基于 TCP，自带可靠重传，不要求无损网络，性能弱于 RoCE，多用于通用服务器、NVMe-oF 兼容场景。

## 三大核心机制

1. 内核旁路（Kernel Bypass）

应用直接在**用户态**通过 Verbs 操作 RNIC 网卡，不经过操作系统内核 socket 协议栈。 省去系统调用、上下文切换、内核缓冲区拷贝，大幅降低时延与 CPU 占用。

2. 远程直接内存访问（Remote DMA）--- CPU零拷贝

网卡硬件自带独立 DMA 引擎，可直接读写对端**注册锁页用户内存 (MR)**。 不需要两端 CPU 搬运数据，彻底消除「内核→用户」CPU 内存拷贝。 分为单边 Write/Read、双边 Send/Recv，大模型张量同步大量使用**单边操作**。

3. 硬件协议卸载（Offload）

报文分片、ACK 确认、重传、流控、CRC 校验全部由 RNIC 硬件完成，不消耗主机 CPU 算力。 传统 TCP 所有逻辑靠 CPU 内核处理，RDMA 把完整传输协议下沉网卡硬件。



## 主流落地业务场景

1. **AI 分布式训练 / 推理（最大增量场景）** 多机梯度 AllReduce、模型并行张量交换、分布式 KV 缓存同步，RoCE+GPUDirect RDMA 标配；
2. **分布式存储 NVMe-oF** RDMA 作为标准传输层，远端 SSD 远程挂载，配合 O_DIRECT/GDS 实现硬盘→GPU 显存全链路零拷贝；
3. **金融高频交易** 微秒级时延，低 CPU 开销；
4. **云渲染 / VR 资产同步、长距云边协同** 智算中心到边缘 AI-RAN 机房光纤长距 RDMA 下发世界模型；
5. **远期：6G W-RDMA、工业 5G 专网** 无线层增加 FEC、SR 选择重传，把 RDMA 延伸至无线边缘节点。


## 🎯 今日总结

**Q1：RDMA核心原理与优势是什么？** A1：网卡硬件直接读写远端用户态内存，内核旁路，实现零拷贝、低时延、极低CPU开销。

**Q2：RDMA三大核心机制是什么？** A2：内核旁路、远程DMA用户内存直访、全协议硬件卸载，从硬件层面彻底解放主机CPU。

**Q3：RDMA完整工作流程是什么？** A3：用户态注册MR锁页内存，通过QP队列下发工作指令，RNIC网卡硬件DMA自主完成数据传输。

**Q4：RDMA有哪三种主流制式、适用场景？** A4：IB（专用无损超算）、RoCEv2（以太网AI集群主流）、iWARP（基于TCP，兼容性强）。

**Q5：原生RoCE最大短板是什么？如何优化？** A5：默认GBN回退重传，微丢包即吞吐雪崩；新一代DPU硬件支持SR选择重传，大幅优化抗丢包能力。

**Q6：AI训练如何依托RDMA实现高性能通信？** A6：GPU通过PCIe对接RNIC网卡，依托RDMA单边读写，完成跨机张量、梯度零拷贝同步。

**Q7：RDMA如何赋能分布式存储？** A7：NVMe-oF 结合RDMA实现硬件DMA接力，可完成硬盘到GPU显存端到端无CPU拷贝传输。

**Q8：RDMA未来技术演进方向是什么？** A8：有线RoCE已商用成熟，未来依托6G+FEC+SR优化，落地W-RDMA，支撑云边AI协同场景。

**RDMA 的 DMA，核心目的是实现极致高速、低时延的跨机网络通信，降低 CPU 占用只是配套手段，而非设计初衷**。
